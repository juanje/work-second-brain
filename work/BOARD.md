# Board

Sections are ordered for **actionability**: what you are doing now, what is next, what is blocked, then sprint scope, capture, tech debt, and finally completed work (mostly for archival — summaries come from the agent, not daily reads).

- **Doing** — active work in progress. **WIP target: 1.** At most **2** items if they are tightly related (same theme, minimal context switching). Avoid parallel unrelated work.
- **Next Actions** — **max 3–4 items.** Immediate queue after Doing; not the full sprint.
- **Waiting** — blocked on someone or something; check if you can unblock or resume.
- **Sprint Backlog** — committed for the current sprint, triaged. Pull into Next Actions when there is capacity. Your issue tracker remains the system of record.
- **Inbox** — capture first, untriaged. Move to Sprint Backlog, Next Actions, Parked, or discard during standup or review.
- **Parked** — ideas, tech debt, nice-to-haves. For when you are less busy.
- **Done** — completions. Cleared during weekly review into review files. You rarely need to read this section; ask the agent for summaries.

Flow: **Capture → Inbox** (triage) **→ Sprint Backlog or Parked** → pull into **Next Actions** (max 3–4) **→ Doing → Done** (Waiting when blocked).

Item format:

```
- [ ] Short description | added: YYYY-MM-DD
  - Context: why this matters, what it blocks, etc.
```

**One actionable unit = one board item.** Each distinct task or ticket gets its own checkbox line. Never fold a second task into another item as a "Related" note — that hides real WIP and breaks tracking. If two items are linked, add a **Related to:** line under each item pointing to the other (both directions).
